# Assets README

Canonical subsystem readme for `assets`.

## Purpose

Holds static and generated project assets used by UI, build, and media pipelines.

## Current State

- Keep assets grouped by domain (audio, video, images, docs, samples).
- Avoid storing executable code in this directory.

## Growth Opportunities

- Introduce lifecycle labels to distinguish source assets from generated or archival artifacts.
